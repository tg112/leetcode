# Daily Temperatures

### 1. 単調スタック（Monotonic Stack）の概念
スタックの中身が常に「単調減少（または単調増加）」の順序を保つように制御するテクニックです。
今回の実装では、スタックの底からトップに向かって「気温が低くなっていく（単調減少）」状態が維持されています。
- 「自分より右側（未来）にある、自分より大きな最初の要素」を見つけたいときに、最強の威力を発揮するデータ構造です。

### 2. 「値」ではなく「インデックス（位置）」をスタックに積む
スタックに気温（例：73度）をそのまま積んでしまうと、「何日後にその気温になったか」という**距離（日数）の計算**ができなくなります。
コード内の `stack.append(i)` のように、インデックスを保存しておくことで、`stack[-1] - i` （未来の暖かい日の位置 - 今日の位置）という一撃の引き算で日数を導き出すことができます。

### 3. 未来から過去へ遡る「逆順走査」のメリット
配列を後ろ（`n-1`）から前に向かって走査することで、「すでにチェックし終わった未来の気温」がスタックに蓄積された状態を作れます。
自分より低い気温のインデックスは `while` ループで次々と `pop()`（排除）されていくため、スタックのトップ（`stack[-1]`）には常に「自分より高くて、一番近い未来の日」が残る仕組みになっています。

### 計算量
- 時間計算量: O(N) — 上述の通り、各要素がスタックにプッシュ・ポップされる回数は高々1回ずつのため、要素数 N に対して O(N) の最速で動作します。
- 空間計算量: O(N) — 最悪のケース（気温が毎日下がり続ける場合など）、すべてのインデックスがスタックに溜まっていくため、最大で N 個分のメモリ（スタック用）を使用します。
