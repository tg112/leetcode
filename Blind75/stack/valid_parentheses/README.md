# Valid Parentheses

## Point

### 1. スタック（Stack）データ構造の「LIFO」特性

スタックは「Last In, First Out（後入れ先出し）」の性質を持つデータ構造です。括弧のペアチェックにおいて、「最後に開いた括弧が、最初に閉じられるべき」というルール（入れ子構造）があるため、スタックが特効薬になります。

- Pythonでは通常のリスト（`[]`）を使って、`append()` でプッシュ、`pop()` でポップを効率よく（O(1) で）行えます。

### 2. ハッシュマップ（辞書）による判定の共通化
`bracket_map = {")": "(", "]": "[", "}": "{"}` のように、「閉じ括弧をキー、対応する開き括弧を値」として定義するテクニックです。

- `if char in bracket_map:` と判定することで、「今見ている文字が閉じ括弧かどうか」を一発で判別できます。
- 今後、括弧の種類が増えたり、別のペア（例：HTMLタグの開始と終了など）に変わったりしても、この辞書の中身を書き換えるだけで同じロジックが使い回せます。

### 3. エッジケース（例外的な入力）への安全弁

スタックの問題で最も起きやすいバグは、スタックが空なのに要素を取り出そうとしてエラー（`IndexError`）になることです。
提示されたコードでは、`if stack and stack[-1] == ...` と判定しており、必ず「スタックが空ではないこと（`stack`）」を先に確認してから末尾の要素（`stack[-1]`）を見に行っているため、非常に安全で堅牢なコードになっています。

## 計算量

- 時間計算量: O(N) — 文字列 s の各文字を左から右へ1回ずつ順番に走査するだけです。スタックへの追加（append）や削除（pop）はすべて O(1) で行われるため、全体の処理時間は文字列の長さ N に完全に比例します。
- 空間計算量: O(N) — 最悪のケース（例：s = "(((((((" のように開き括弧だけが連続する場合）、文字列のすべての文字がスタックに格納されるため、最大で N 個分のメモリを消費します。
