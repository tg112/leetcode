# Valid Parentheses

## 問題概要
`(`, `)`, `{`, `}`, `[`, `]` からなる文字列 `s` が与えられる。すべての括弧が正しい順序・対応で閉じられているかを判定する。

## 計算量
- 時間計算量: O(n) — 文字列を左から右へ1回走査。`append`/`pop` はすべて O(1)
- 空間計算量: O(n) — 最悪ケース（`"((((("` のように開き括弧だけ）でスタックが n 個分のメモリを消費

## アプローチ

**Stack（LIFO）** を使う。「最後に開いた括弧が最初に閉じられるべき」という入れ子のルールと LIFO の性質が完全に一致する。

1. 開き括弧 → スタックに push
2. 閉じ括弧 → スタックの top が対応する開き括弧と一致すれば pop、不一致なら即 `False`
3. 全文字処理後、スタックが空 → `True`、残っていれば `False`

## ポイント

**ハッシュマップで判定を共通化**  
`bracket_map = {")": "(", "]": "[", "}": "{"}` と定義することで:
- `if char in bracket_map:` → 閉じ括弧かどうかを一発で判別
- `bracket_map[char]` → 対応する開き括弧を O(1) で取得
- 括弧の種類が増えても辞書の中身を変えるだけでロジックを使い回せる

**スタック空チェックの安全弁**  
`if stack and stack[-1] == bracket_map[char]:` のように、`stack` が空でないことを先に確認してから `stack[-1]` を参照することで `IndexError` を防ぐ。
