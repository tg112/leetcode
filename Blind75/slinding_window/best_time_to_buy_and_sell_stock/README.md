# Best time to buy and sell stock

- Time Complexity: O(N) 
    - 配列 prices の要素を最初から最後まで1回だけスキャするため、株価の日数 N に対して比例した時間で終わります。
- Space Complexity: O(1)
    - 最小価格（min_price）と最大利益（max_profit）を記録する2つの変数しか使わないため、どれだけ配列が長くなっても追加のメモリを消費しません。

## Point

- 過去の最安値を記憶しながら、未来の最高利益を更新していく
- 株価が下がった日」は、売っても利益が出ない（または前より少なくなる）ことが確定しているため、利益の計算（else 側）をスキップ
- 1つの配列に対する動的計画法（DP）の簡易版 / 貪欲法
    - これまでの最小値（または最大値）をキープしながら進む
    - 時系列データの中で、過去の特定の状態に基づいて現在の最適な行動を決める」問題全般に使える
