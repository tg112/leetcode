# Longest Substring Without Repeating Characters（

## Point

文字が絶対に被らないように、*文字の『窓』を右へ右へと広げていった*とき、最大でどれだけ大きくできるか

`char_index_map[s[right]] >= left` の条件がある理由

ex) 例えば、"abba" という文字列を処理する場合を考えます。

1. a, b, b と進んだとき、b が重複するので left は2番目の b の位置（インデックス2）まで進みます。このとき、過去の a（インデックス0）はすでに窓の外（左側）に置き去りになっています。
2. 最後に4文字目の a に到達したとき、ハッシュマップは過去の a（インデックス0）を覚えています。
3. もし上記の条件式がないと、せっかく進めた left が、すでに過去のものとなったインデックス0のせいで後ろ（左）に逆戻りする。
4. char_index_map[s[right]] >= left と判定することで、「すでに窓の外にある古い重複は無視する」という安全弁が働き、バグを防ぐ。

## 計算量

時間計算量: `O(N)` — 文字列を right ポインタが左から右へ1回スキャンするだけで完了します（left の移動にループを消費しないため、文字の戻りがありません）。空間計算量: `O(min(m, n))` — ハッシュマップに登録されるのは文字列に登場するユニークな文字だけです。文字列の長さを `n`、扱いうる文字の種類（アルファベットや記号の総数）を `m` とすると、その小さい方に依存するため、実質 `O(1)` のメモリ効率と言えます。
