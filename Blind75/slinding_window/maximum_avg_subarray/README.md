# Maximum Average Subarray

## Point

- 最初にベースとなる窓の合計を作る(`sum(nums[:k])` を使って、合計値を計算する)
- 平均の計算は最後に一回だけ行う

## Sliding Window Pattern

スライディングウィンドウとは、配列や文字列の「連続する一定範囲（窓）」をズラしながら効率的に処理する手法です。

### なぜこのパターンを使うのか？（累積和の再利用）

もし長さ k の部分配列の合計を毎回愚直に計算 `（sum(nums[start:end])）` すると、窓を1つ進めるたびに `O(k)` の計算がかかり、全体で `O(N X k)` になってしまいます。しかし、窓が右に1つズレる時、「変化しているのは両端の2つの要素だけ」という点に着目します。

*捨てるもの*： 窓から外れる「左端の要素」  
*入れるもの*： 新しく窓に入る「右端の要素」  
*それ以外*： 真ん中の要素たちの合計はそのまま使い回せる！

この「引き算1回、足し算1回」で次の窓の合計を出す仕組みにより、計算量を `O(N)` に劇的に削減できます。
