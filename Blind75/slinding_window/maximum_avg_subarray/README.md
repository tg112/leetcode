# Maximum Average Subarray

## 問題概要
整数配列 `nums` と整数 `k` が与えられる。長さ `k` の連続する部分配列の中で、平均値が最大のものを返す。

## 計算量
- 時間計算量: O(n) — 窓を右にスライドするループ1回
- 空間計算量: O(1) — 合計値を保持する変数のみ

## アプローチ

**固定サイズ型スライディングウィンドウ** を使う。

1. 最初の窓（インデックス `0` から `k-1`）の合計を計算してベースにする
2. 窓を1つ右にズラすたびに:
   - 左端（外れる要素）を引く
   - 右端（入る要素）を足す
3. 「引き算1回 + 足し算1回」で次の窓の合計が得られる
4. 最後に `max_sum / k` で平均を返す

## ポイント

**なぜ O(n×k) → O(n) になるのか**  
愚直に毎回 `sum(nums[i:i+k])` を計算すると O(k) かかり全体で O(n×k) になる。  
しかし窓が1つ右にズレるとき、変化するのは両端の2要素だけ。真ん中の要素の合計はそのまま使い回せるため、O(1) で次の窓の合計を得られる。

**平均の計算は最後に1回だけ**  
ループ中は合計値のみを比較し、除算を最後の1回に絞ることで計算コストを最小化する。
