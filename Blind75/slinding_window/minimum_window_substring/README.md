# Minimum Window Substring

## 問題概要
文字列 `s` の中から、`t` に含まれるすべての文字（重複を含む）を包含する最短の部分文字列を返す。存在しない場合は空文字 `""` を返す。

## 計算量
- 時間計算量: O(|S| + |T|) — `t` のカウントマップ作成に O(|T|)、`s` の走査に O(|S|)
- 空間計算量: O(|S| + |T|) — 2つのハッシュマップ（文字の種類が限定される場合は実質 O(1)）

## アプローチ

**収縮型スライディングウィンドウ** を使う。

- `dict_t`：`t` の各文字の必要数（固定）
- `window_counts`：現在の窓内の各文字の出現数（動的）
- `formed`：現在の窓で条件を満たしたユニーク文字の種類数
- `required`：`t` のユニーク文字の種類数（ゴール）

1. **拡大フェーズ**：`formed < required` の間、`right` を右に進める
2. **縮小フェーズ**：`formed == required` になったら、`left` を右に動かして窓を最小化し、最小長を記録する
3. `right` を進めて繰り返す

## ポイント

**`formed` の更新は「境界をまたいだ瞬間」だけ**
- `window_counts[char] == dict_t[char]` になったとき → `formed += 1`
- `window_counts[char] < dict_t[char]` になったとき → `formed -= 1`
- `<=` や `>` ではなく `==` と `<` の閾値を使うことでバグを防ぐ

**文字列の切り出しは最後の1回だけ**  
ループ中に `s[left:right+1]` を毎回切り出すと O(n) のコピーが走り TLE になる。  
`min_left` と `min_right` でインデックスだけを記録し、最後に1度だけ `s[min_left:min_right+1]` を切り出すのが鉄則。
