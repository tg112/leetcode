# Longest Repeating Character Replacement

## 問題概要
大文字英字のみからなる文字列 `s` と整数 `k` が与えられる。最大 `k` 文字を任意の文字に書き換えたとき、同じ文字が並ぶ最長の部分文字列の長さを返す。

## 計算量
- 時間計算量: O(n) — `right` と `left` がそれぞれ最大1回ずつ配列を走査
- 空間計算量: O(1) — ハッシュマップのキーは英大文字最大26種で固定

## アプローチ

**可変サイズ型スライディングウィンドウ** を使う。

条件：`(窓の長さ) - (窓内の最頻出文字数) <= k`  
→ 窓の中で「最多の文字」以外をすべて書き換えるコストが k 以内

1. `right` を右に伸ばし、文字カウント (`count`) と `max_frequency` を更新
2. 条件を超えたら `left` を右に縮め、左端の文字カウントを減らす
3. 常に `max_len = max(max_len, right - left + 1)` で最大長を記録

## ポイント

**固定サイズ型との違い**  
Maximum Average Subarray は窓サイズ `k` が固定だが、この問題は条件を満たす限り右端を伸ばし、満たさなくなったら左端を縮める「サイズが伸び縮みする窓」を使う。

**`max_frequency` を再計算しない理由**  
左端を縮めても `max_frequency` をわざわざ更新しない。なぜなら、`max_frequency` が過去の最高記録を下回っているときの窓は、どうせ最大長を更新できないからスキップしても問題ないから。
