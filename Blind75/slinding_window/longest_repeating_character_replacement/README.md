# Longest Repeating Character Replacement

## Point

文字を `k` 個まで書き換えていいから、同じ文字がずらっと並んだ一番長い区間（窓）を作る

1. スライディングウィンドウ（可変サイズ型）これまで扱ってきた「Maximum Average Subarray」は窓のサイズが `k` で固定でしたが、今回は条件を満たす限り右端を伸ばし、満たさなくなったら左端を縮めるという「サイズが伸び縮みする窓」の知識が必要です。
2. ハッシュマップ（count）による文字数のリアルタイム管理右端が進むたびに文字数をカウントアップし、左端が削られるたびにカウントダウンします。これにより、「今、窓の中に何の文字がいくつあるか」を常に正確に把握します。3. max_frequency の面白い特性（賢いサボり方）本来、左端を削った（left += 1）ときは、窓の中の文字数が変わるため max_frequency を再計算（ハッシュマップ全体の最大値を再確認）するべきに見えます。しかし、このアルゴリズムでは再計算しなくてOKです。なぜなら、私たちが探しているのは「過去最大の長さ」なので、max_frequency が過去の最高記録を下回るような小さな窓のときは、どのみち最大記録を更新できないからです。

## 計算量

 - 時間計算量: `O(N)` — right ポインタが配列を1周し、left ポインタも最大で配列を1周するだけなので、文字列の長さ `N` に対して線形時間で終わります。
 - 空間計算量: `O(1)` — ハッシュマップ count に入るキーは、英大文字のみであれば最大26個で固定されるため、文字列がどれだけ長くなってもメモリは一定です。
