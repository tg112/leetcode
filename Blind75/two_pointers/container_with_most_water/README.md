# Container With Most Water

## 問題概要
整数配列 `heights` が与えられる（`heights[i]` は i 番目の棒の高さ）。任意の2本の棒を選んで作るコンテナが蓄えられる最大の水の量を返す。

## 計算量
- 時間計算量: O(n) — `left` と `right` が合わせて最大 n 回移動
- 空間計算量: O(1)

## アプローチ

**Two Pointers（貪欲法）** — 両端から挟み、低い方の壁を動かす。

1. `left = 0`、`right = n-1` に初期化
2. 面積 = `min(heights[left], heights[right]) × (right - left)` を計算し `max_area` を更新
3. **低い方の壁のポインタを内側へ移動**して繰り返す

## ポイント

**なぜ「低い方」を動かすのか？**  
幅は1ずつ縮まっていくため、面積を増やすチャンスは「高さを上げること」だけ。ボトルネック（低い壁）を動かさない限り、面積が改善する見込みは全くない。逆に「高い方」を動かすと、高さが下がるかそのままで幅だけ縮まるため確実に損をする。

```
heights = [1, 8, 6, 2, 5, 4, 8, 3, 7]

left=0(高さ1), right=8(高さ7) → 幅8 × 高さ1 = 面積8
低い左(1)を動かす → left=1

left=1(高さ8), right=8(高さ7) → 幅7 × 高さ7 = 面積49 ← 最大
```
